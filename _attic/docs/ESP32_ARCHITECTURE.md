# BookCabinet — Архитектура ESP32

## Обзор

Система разделена на два уровня:
- **RPi** — бизнес-логика, API, UI
- **ESP32** — механика, автономное управление

## Компоненты

| Компонент | Количество | Тип |
|-----------|------------|-----|
| CoreXY моторы (A, B) | 2 | NEMA17 шаговые |
| Мотор платформы | 1 | NEMA17 шаговый |
| Концевики каретки | 4 | X_BEGIN, X_END, Y_BEGIN, Y_END |
| Концевики платформы | 2 | LIFT_TOP, LIFT_BOTTOM |
| Концевик лотка | 1 | TRAY |
| Сервы-замки | 2 | PWM |
| Реле шторок | 2 | Внутренняя + внешняя |

## ESP32 DevKit — Распиновка

```
# Моторы (6 пинов)
GPIO 12 → A_STEP
GPIO 14 → A_DIR
GPIO 27 → B_STEP
GPIO 26 → B_DIR
GPIO 25 → LIFT_STEP
GPIO 33 → LIFT_DIR

# Концевики (7 пинов)
GPIO 32 → X_BEGIN
GPIO 35 → X_END
GPIO 34 → Y_BEGIN
GPIO 39 → Y_END
GPIO 36 → LIFT_TOP
GPIO 4  → LIFT_BOTTOM
GPIO 15 → TRAY

# Сервы (2 пина)
GPIO 13 → SERVO_LOCK_1
GPIO 2  → SERVO_LOCK_2

# Реле (2 пина)
GPIO 17 → RELAY_SHUTTER_IN
GPIO 16 → RELAY_SHUTTER_OUT
```

## Разделение ответственности

### RPi (бизнес-логика)
- API для внешнего мира
- Решает какую книгу выдать/принять
- Учёт, логи, UI, интеграции
- **НЕ знает про моторы и координаты**

### ESP32 (механика)
- Координаты всех полок (хранит в памяти)
- Алгоритмы движения, выдачи, приёма
- Homing, safety, концевики
- Управление замками, шторками
- **Автономен** — завершит операцию даже если RPi упал

## Протокол (JSON через Serial/USB)

### RPi → ESP32 (команды)

```json
{"cmd": "give", "shelf": 5}
{"cmd": "take", "shelf": 3}
{"cmd": "home"}
{"cmd": "status"}
{"cmd": "lock", "id": 1, "state": "open"}
{"cmd": "shutter", "id": "inner", "state": "open"}
```

### ESP32 → RPi (обратная связь)

```json
{"ack": "give", "shelf": 5}
{"progress": "moving_to_shelf", "shelf": 5, "x": 1500, "y": 3200}
{"progress": "waiting_for_book", "shelf": 5}
{"done": "give", "shelf": 5, "success": true}
{"error": "book_not_taken", "shelf": 5, "timeout": 30}
{"status": "idle", "x": 0, "y": 0, "lift": 0, "locks": [0,0], "shutters": [0,0]}
```

## Состояния ESP32

| Состояние | Описание |
|-----------|----------|
| `idle` | Ожидание команды |
| `homing` | Калибровка (поиск концевиков) |
| `moving` | Движение к позиции |
| `waiting` | Ждёт книгу/действие пользователя |
| `error` | Ошибка (требует вмешательства) |

## Калибровка (данные от 03.03.2026)

### Безопасные границы
```
X: б0 = -1000, бmax = -20000 (от реального 0)
   Диапазон: 19000 шагов
   
Y: б0 = -4000, бmax = +41000 (от реального 0)
   Фактический диапазон: 45000 шагов
```

### CoreXY направления
```
Вверх:   A_DIR=1, B_DIR=0
Вниз:    A_DIR=0, B_DIR=1
Влево:   A_DIR=0, B_DIR=0
Вправо:  A_DIR=1, B_DIR=1
```

### Диагональное движение
```
A_steps = X_steps + Y_steps
B_steps = X_steps - Y_steps
```

## Преимущества архитектуры

- Одна команда = вся операция (не много мелких)
- ESP32 автономен
- Простая отладка (можно тестировать без RPi)
- Легко масштабировать (новый шкаф = новый конфиг в ESP32)
